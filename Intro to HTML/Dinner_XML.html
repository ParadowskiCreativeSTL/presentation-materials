HTML is just a specialized form of XML, so clients that consume the html document (clients that consume the html doument is just a fancy way of saying "internet browsers, certain text editors, etc. Stuff that will open your html file so you can see it or use it") will still be able to read the xml data.

The reason we're only seeing the content of the elements is because the element names we're using aren't ones that browsers understand.

Any client that can consume HTML knows that certain html elements should be drawn on the screen in specific ways. Elements that the client doesn't understand just renders out.
